# 2.6 Настройка технологического обеспечения

Перед началом непосредственной разработки информационной системы необходимо выполнить настройку технологического обеспечения, окружения разработки и инструментов, которые используются для запуска клиентской, серверной и аналитической частей проекта.

Разрабатываемая система представляет собой платформу контроля качества медиаконтента Media Quality Control Platform. В состав проекта входят пользовательский интерфейс, серверное API, сервис автоматического анализа медиаматериалов, база данных и объектное хранилище файлов. Поэтому технологическое обеспечение должно поддерживать одновременную работу нескольких компонентов и обеспечивать воспроизводимый запуск системы на разных устройствах.

В качестве интегрированной среды разработки был выбран Visual Studio Code. Данный инструмент подходит для разработки веб-приложений на TypeScript и Python, поддерживает работу с терминалом, Git, Docker, расширениями для NestJS, React, Prisma и Python, а также позволяет удобно работать с несколькими частями проекта в одной рабочей области.

В качестве основной платформы для клиентской и серверной части используется Node.js. Node.js позволяет выполнять JavaScript и TypeScript-код вне браузера, а также предоставляет менеджер пакетов npm, который применяется для установки библиотек и запуска сценариев разработки. Для корректной работы проекта необходима версия Node.js 20 или выше.

После установки Node.js корректность установки проверяется в терминале Visual Studio Code следующими командами:

- `node -v`;
- `npm -v`.

Отдельным компонентом проекта является сервис автоматического анализа медиаконтента `analyzer`. Он реализован на Python с использованием FastAPI. Данный сервис отвечает за автоматическую проверку загруженных материалов и передает результаты проверки в серверную часть.

Для локальной настройки analyzer необходимо создать виртуальное окружение Python и установить зависимости:

```bash
cd analyzer
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
```

В проекте PostgreSQL запускается через Docker Compose на основе образа `postgres:16-alpine`.

- внутренний порт контейнера - `5432`;

Одним из основных конфигурационных файлов Node.js-приложений является `package.json`. В данном проекте отдельные файлы `package.json` находятся в директориях `frontend` и `backend`. Они содержат имя приложения, версию, список зависимостей, зависимости для разработки и доступные команды запуска. При установке зависимостей также создается файл `package-lock.json`, который фиксирует точные версии установленных пакетов и обеспечивает одинаковую установку зависимостей на разных компьютерах.
